Begreppsförklaring

Element:

Ett element i HTML är en del av ett HTML dokument som börjar med en starttagg och avslutas med en sluttagg. Allt ifrån ord till bilder och färger kan tillägas med hjälp av olika element. Några exempel är h1, h2, img src.

HTML:

Står för HyperText Markup Language, är del av den grundläggande standarden för World Wide Web. De flesta hemsidor skrivs i HTML och överförs med HTTP.

ID:

Är ett sätt att döpa ett element i html. Som en class fast ID gör elementet unikt. Detta gör att man bara kan använda ID en gång per HTML. Med hjällp av ID:et kan man ändra saker i elementet i css till exemplel.

Class:

Man kan använda sig av classes i html för att namnge ett element och sedan på ett lätt sätt kunna ändra på detta i en css fil till exempel.

Meta:

Meta taggen skriver man inuti headen och dessa taggar bestämmer, teckenuppsättning, sidbeskrivning, nyckelord och ägaren över dokumentet. Dessa taggar synd inte på sidan utan är information som jobbar i bakgrunden och på hela sidan.

Responsive:

Gör att man innehåll ändras beroende på webbläsares olika storlekar, bilder ska t.ex. inte ta upp hela skärmen om man använder mobilen t.ex. Texten ska också ändras efter olika storlekar.

HTTP:

Står för Hypertext Transfer Protocol, detta används för att föra över sidor från World Wide Web till Internet. Bygger på kommunikationen mellan klient och server, där förfrågan och svar jobbar med varandra.

HTTPS:

Står för Hypertext Transfer Protocol Secure och används för att kryptera data från ett http dokument. Detta brukar användas vid betalningar eller behandling av känslig information. Detta gör datatransporten mycket säkrare.

HTTPS:

Står för Hypertext Transfer Protocol Secure och används för att kryptera data från ett http dokument. Detta brukar användas vid betalningar eller behandling av känslig information. Detta gör datatransporten mycket säkrare.

Compressed data:

Att komprimera data betyder att man ändrar, kodar om strukturen av data för att datan ska ta upp mindre plats. Detta gör att filstorlekar minskar och att man kan lagra mer data med samma diskstorlek.

CDN:

Står för Content Delivery Network och minskar avståndet mellan användare och servern, detaa gör att det går mycket snabbare att ladda en sida till exempel. Utan detta måste ursprungliga servrar svara varenda användare och förfrågan, detta blir tungt och kan leda till förlust av data.

Local bootstrap:

Detta gör att man ladda saker snabbare på en sida eftersom det använder sig av CDN. Detta kan användas för att göra en hemsida snabbare och ge användarna en bättre upplevelse.

JQuery:

Är det poluläraste biblioteket på webben idag. Detta innehåller javascript funktioner förenklar det för utveklare att göra vanliga saker som til exempel göra effekter eller animationer.

Flexbox:

Är en layoutmodell som tillåter en att justera och fördela mellanrum innuti en container, med hjälp av bredder och höjder kan man fylla en sida, och är bra att använda för att göra olika elements responsive.

Grid:

Med hjälp av grid kan man dela upp en sida i olika delar, detta gör att man kan justera element enkelt. Man kan enklare designa en sida utan att behöva positionera saker och använda floats lika mycket.
